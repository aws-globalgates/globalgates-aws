GlobalGates에 넣을 AI 콘텐츠 후보

현재 DB에는 회원, 게시글, 상품, 좋아요, 북마크, 댓글, 커뮤니티, 신고, 검색 이력이 있어 회귀형 점수 예측에 바로 쓸 수 있습니다.

1

AI 추천 적합도 점수

  • 상품/게시글이 특정 사용자에게 얼마나 유용할지 0-100점으로 예측
  • 추천제품, 메인 피드, 커뮤니티 피드 정렬에 사용
  • 기존 최신순 추천의 불편함을 바로 줄임
2

거래/문의 가능성 점수

  • 상품이 채팅, 견적, 북마크로 이어질 가능성을 예측
  • 구매자는 반응 좋은 상품을 빨리 찾고, 판매자는 글 품질을 개선
  • 채팅/견적 완료 데이터가 충분해지면 정확도 상승
3

가격 적정성 가이드

  • 카테고리, 제목, 태그, 과거 상품 가격으로 예상 가격을 회귀 예측
  • 판매자에게 "비슷한 상품 대비 높음/낮음" 표시
  • 브랜드, 상태, 거래완료 가격 데이터가 없으면 보조 기능으로 시작
4

검색 결과 품질 점수

  • 검색어와 게시글/상품의 클릭 가능성, 저장 가능성을 예측
  • 관련 검색어와 인기순 정렬을 더 정교하게 개선
  • 검색 노출/클릭 로그를 추가하면 효과가 커짐
5

커뮤니티 성장 예측

  • 커뮤니티의 다음 7일 가입/게시글 증가량을 예측
  • 탐색 화면에서 활발한 커뮤니티를 먼저 보여줌
  • 초기 데이터가 적으면 규칙 기반과 섞어 사용
6

운영 위험도 점수

  • 예상 신고 수 또는 블라인드 가능성을 회귀 예측
  • 사용자 피드 품질과 운영자 처리 우선순위 개선
  • 자동 제재가 아니라 관리자 참고 점수로 시작

1차 MVP: AI 추천 적합도 회귀 모델

사용자에게 보이는 콘텐츠

  • 추천제품 카드에 "AI 추천 87점", "북마크 가능성 높음", "관심 카테고리와 가까움" 배지 표시
  • 기존 최신순 상품 목록을 예측 점수 높은 순서로 정렬
  • 점수가 낮거나 신고가 많은 글은 뒤로 밀고, 차단/신고 필터는 기존처럼 유지

왜 회귀가 맞는가

분류는 "좋다/나쁘다"처럼 나누지만, 추천 정렬은 숫자 점수가 필요합니다. 좋아요, 댓글, 북마크, 조회, 신고를 합쳐 목표 점수를 만들고, 새 상품이 들어왔을 때 예상 점수를 회귀로 예측하면 화면 정렬에 바로 쓸 수 있습니다.

target_score = 1.0 * log1p(view_count_7d) + 2.0 * like_count_7d + 3.0 * bookmark_count_7d + 2.5 * reply_count_7d - 5.0 * report_count_7d

학습 피처 설계

상품/게시글 피처 가격, 재고, 카테고리, 제목 길이, 내용 길이, 이미지 수, 해시태그 수, 작성 후 경과 시간, 커뮤니티 여부
작성자 피처 가입 후 경과일, 팔로워 수, 팔로잉 수, 최근 게시글 수, 배지 여부, 신고 누적 수
사용자-상품 피처 사용자의 관심 카테고리 일치 여부, 같은 작성자 팔로우 여부, 같은 지역 여부, 최근 검색어와 제목 유사도
품질/안전 피처 신고 수, 차단 관계 제외, 비활성 게시글 제외, 너무 오래된 글 감점, 재고 0 제외 또는 감점

추천제품 화면 목업

현재 `explore.html`의 추천제품 탭에 붙일 수 있는 형태입니다. 핵심은 사용자가 "왜 이 상품이 먼저 보이는지"를 짧은 배지로 이해하게 하는 것입니다.

화면에 추가되는 요소

  • 1. 상단 정렬 상태 추천제품 탭 우측 또는 카드 목록 상단에 `AI 추천순` 배지를 표시합니다.
  • 2. 카드 점수 상품 제목 위 또는 바로 아래에 `AI 추천 91점`을 붙입니다.
  • 3. 추천 사유 `관심 카테고리`, `북마크 가능성 높음`, `가격 반응 양호`처럼 2개까지만 표시합니다.
  • 4. 기존 액션 유지 좋아요, 조회수, 북마크 버튼은 그대로 두고 점수는 정렬과 설명에만 사용합니다.
  • Fallback FastAPI가 내려가거나 점수가 없으면 최신순 목록과 기존 카드 UI로 돌아갑니다.

구현 흐름

1. 데이터셋 생성 PostgreSQL에서 상품/게시글/반응/신고/작성자 피처를 CSV로 추출
2. Jupyter 학습 `c_regression`의 workflow를 복사해 Linear/Ridge/DecisionTree/RandomForest 비교
3. 검증 R2, MAE, RMSE 확인. 점수 구간별 실제 반응률도 같이 확인
4. FastAPI 서빙 `model.pkl`을 로딩하고 `/predict/product-fit`로 점수 반환
5. Spring 연동 추천 후보 50개를 조회한 뒤 FastAPI 점수로 재정렬하고 화면에 배지 표시

Spring Boot 변경 지점

파일 변경 내용 검증
`PostProductMapper.xml` 추천 후보를 최신순 20개가 아니라 50-100개 가져오고, 피처 컬럼을 함께 조회 기존 추천 상품 목록이 깨지지 않는지 확인
`PostProductDTO.java` `aiScore`, `aiReasons` 필드 추가 JSON 응답에 점수/사유가 포함되는지 확인
`PostProductService.java` FastAPI 클라이언트 호출 후 `aiScore desc` 정렬 FastAPI 장애 시 기존 최신순으로 fallback
`explore/layout.js` 카드에 `AI 추천 87점`, 사유 배지 렌더링 점수가 없으면 배지를 숨김
