핵심 방향

회귀는 라벨을 고르는 게 아니라 연속된 숫자(가격·일수·클릭수·금액·확률점수)를 맞히는 모델입니다. GlobalGates의 상품 거래·견적·광고·게시글·결제 흐름에서 사용자가 "얼마? 며칠? 몇 명?"을 궁금해하는 자리에 숫자를 표시해, 입력 자체가 보조 정보가 되도록 설계했습니다.

서비스 스캔 11개 상품·견적·광고·게시글·알림·결제·구독·회의·관리자·뉴스·커뮤니티 회귀 자리
외부 사례 13개 당근·Aliexpress·Amazon·Airbnb·Uber·Zillow·YouTube·Pinterest·LinkedIn·Coupang 등
모델 후보 10개 시세·ETA·견적응답·광고클릭·도달·매출·이탈일·CPC·회의시간·SLA
제외 분류·LLM 신고 라벨링·텍스트 생성·외부 API 호출형은 분류 계획안 또는 별도 문서

우리 사이트에 먼저 넣을 회귀 AI 콘텐츠

1. 상품 등록 보조

  • 적정 등록가 추천: 카테고리·재고·지역·과거 거래가 기반 권장가와 신뢰구간 (당근 시세 모티브)
  • 거래완료 예상일: "이 가격대·카테고리·지역은 평균 3.4일 안에 마감" (배송지연 LGBM 응용)
  • 예상 조회수/문의수: 이미지 수·태그·본문 길이 기반 입력 보조 카드

2. 견적/광고 의사결정 보조

  • 견적 응답 예상시간: 카테고리·지역·전문가 풀 기반 "평균 4시간 내 응답"
  • 광고 예상 클릭/도달: 예산·헤드라인·랜딩 도메인 기반 회귀 → AdvertisementVO.impressionEstimate 자리에 직접 표시
  • 예산 소진 예상일: 일평균 클릭/CPC 회귀로 "남은 예산 6.8일 안에 소진"

3. 운영/리텐션 보조

  • 구독 이탈까지 예상일: 최근 활동·결제 이력 기반 운영자 대시보드
  • 다음 7일 매출 예측: 판매자/광고주 마이페이지에 라인차트
  • 신고 SLA 예측: 누적·심각도·담당자 부하 기반 "예상 처리 22분"

회귀에 좋은 신호 (DB → 피처)

  • 지역: PostVO.location, MemberVO.memberRegion / memberCountry

활동/시간 신호

  • 최근 활동: MemberVO.lastLoginAt, 게시글 수, 채팅 수

  • 결제 이력: tbl_payment 성공/실패 빈도, 구독 갱신 간격
  • 운영 부하: tbl_report 처리 큐 길이, 담당자 수

Jupyter → FastAPI → Spring Boot 연동 구조

작업 폴더는 사용자가 지정한 /Users/yoonchan/Desktop/gb/ai/machine-learning/c_regression 기준입니다. 이미 학습된 delivery_time_lgbm_model.pkl과 freight_cost_linear_model.pkl을 1차 MVP의 ETA/시세 모델 시드로 활용할 수 있습니다.

1. DB 추출 Spring DB에서 게시글, 상품, 광고, 견적, 결제, 활동 로그를 CSV로 추출. 시간 컬럼은 UTC 기준.
2. 노트북 학습 EDA → train/val/test split → 결측 처리 → StandardScaler/log1p → Linear/Ridge/Tree/LGBM 비교 → 잔차 점검
3. 모델 저장 joblib.dump로 pipeline, feature 순서, target transform, metrics를 한 폴더에 묶어 저장
4. FastAPI 서빙 모델 이름과 입력 피처를 받아 예측값 + 신뢰구간 + topFeatures를 반환
5. Spring 적용 상품/광고/견적/관리자 화면에 RestClient로 호출 후 카드/배지로 렌더링. 결과 캐시 30초.

검증 기준

분류는 confusion matrix·F1 위주였지만, 회귀는 평균오차(MAE)·제곱오차(RMSE)·결정계수(R²)·상대오차(MAPE)·잔차 분포로 검증합니다. 화면에 직접 숫자가 보이는 만큼 "체감 오차"가 곧 신뢰 지표입니다.

오프라인 지표

MAE (평균 절대오차)가격: ≤ 1,500원
RMSE가격: ≤ 2,200원
R²≥ 0.75
MAPE≤ 14%
잔차 분포대칭, 이상값 컷오프 ±3σ

운영 검증

  • 화면 노출 전 shadow mode로 1주 운영자 KPI와 비교
  • 가격/ETA처럼 사용자에게 직접 보이는 모델은 신뢰구간 함께 표시
  • 이탈/매출처럼 행동을 트리거하는 모델은 임계값 + 휴먼 인 더 루프
  • 실데이터 분포 변화 감시: 일별 PSI/KS 모니터링, 임계 시 재학습 알림

사용자 효과 KPI

  • 상품 등록 이탈률 ↓ (시세 카드 노출 후)
  • 추천가 적용률 (등록가가 추천 CI 안에 들어오는 비율)
  • 실제 거래완료일과 ETA의 평균 절대오차 (체감 정확도)
  • 견적 요청 → 응답 도달 시간 단축률
  • 이탈/매출 KPI 카드 클릭률, 케어 푸시 전환율

잔차 점검 시각화 (price_recommender 예시)

예측가 구간별 잔차 비율 (회색=정상, 황색=경계, 적색=이상)
대부분 회색이지만 고가 구간에서 황색이 늘어나는 경향 → 고가 상품에는 별도 모델 또는 log1p 변환 검토.

학습 노트북 산출물 체크리스트

파일 내용 왜 필요한가
01_dataset_export.ipynb tbl_post / tbl_post_product / tbl_advertisement / tbl_estimation을 CSV로 추출, 분포 확인 실데이터 기반이라는 근거
